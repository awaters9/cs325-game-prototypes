JetPackers

  • Originality: The mechanic of the game is unique. The sole purpose of the game is to fly around and try to slam the opponent into spikes.
  • Technical Merit: The game will randomly spawn spikes around the borders of the map and work on timers. Spikes have multiple phases: 1) The spike moves to an initial position 2) Waits for a few seconds 3) Fires the spike The game also has a separate timer to retract walls of spikes along the borders of the game. It can randomly choose between how many world bounds will be used. These spikes work similarly to the previously mentioned ones but they do not fire. Instead they detract behind the world bounds after a few seconds. I also put effort towards animating the background, which is also on timer events
  • Prototype Postmortem: I learned an absurd amount of stuff. P2 physics: I wanted to use P2 physics to make the spikes have a proper hitbox. I managed to accomplish this initially but eventually reverted the spikes back to arcade physics. P2 can do a lot of things but it definitely isn't arcade physics. P2 can't check for sprite overlaps and Phaser doesn't like to check for separate physic systems interacting with each other. Also, P2 doesn't have the same properties as arcade. If I switched the player's physics to P2, I would then have to make physic materials and alter some functions to cooperate and I just didn't have the time to really invest this kind of commitment. Timer Events: This saved me so much headache when trying to make objects work on timers. It's awesome that I don't have to write a system that generates new timers and instead I can just add events and set delay times for functions. Polish: I probably focused way too much on making the game look 'good'. I prioritized art over making the game more playable. So there are still the same issues of controllers not working when refreshing the page and the game doesn't have a replay button or win screen.
