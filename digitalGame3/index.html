The First Dino of Summer

  • Originality: Not often you see a Dinosaur attending school hmmm?
  • Technical Merit: Randomly places the score objective and which lane the enemy will spawn on. The world moves faster the more points the player accrues.
  • Prototype Postmortem: I learned about the importance of textures and that fast moving tiles are not comfortable on the eyes. I also got to learn more about javascript and the way that Phaser handles changing scenes. The easiest part to making this project was setting up the environment. The more challenging aspect was learning more of Phaser's unintuitive scoping. Trying to figure out how to add events and why things operate the way they do is always a challenge. This way of game making is a tad on the higher level of things. The tutorial and example code that are provided by Phaser are comical in their usability. It's a library that's supposed to take the headache out of creating games yet the example codes don't provide much application to actual game mechanics. Frustrating.

  • Prototype Closest Other Game: I know there are other games like this that I've played before but the unfortunate bit is how unremarkable they are. The concept alone has been improved upon to what is now known as Temple Run. This game is like Temple Run with a lot of gutted mechanics.
    • The code for this example is adapted from the Phaser Basic Project Template
  • High Concept: A lane runner where a dino collects his final grades and avoids other students to be the first dino to leave school.
  • Theme: State which of the Storymatic themes you used.
  • Mandated Variety: State which input, randomness, genre, and play style you used in this prototype. INPUTS: UP ARROW: Shifts the player UP one lane DOWN ARROW: Shifts the player DOWN one lane RANDOMNESS: Random dinosaurs appear in the lanes to act as obstacles. Graded papers spawn and are there to be collected! PlAY STYLE: This game exploits a user's reaction time. There is also a risk for reward where each time the player collects a paper, the player then moves faster. GENRE: It's a Temple Run kinda game
  • Prototype Goal: This game tests a player's reaction time and game awareness to make quick decisions in order to win.
  • Player Experience Goals: Players should feel like there is a risk for reward. Where the higher the score gets, the more difficult the game becomes.
  • Gameplay: This game only allows players to press 2 keyboard buttons. Players can move up and down where their main goal is to avoid other dinosaurs and pick up papers in order to be the first dinosaur to leave school for summer break. As you collect more papers, the speed increases. Collect 10 pieces of paper to win the game. Avoid getting vertigo!
  • Strategies: Players who can focus and have quick reaction time will do well playing this game.
  • Story/Setting/Premise: In this world, dinosaurs attend grade school. You play as a blue dino who's trying to collect his final grade papers so he can leave school and enjoy his summer break! The gameplay can make for some dramatic moments where the player is moving so fast that it's hard to focus on him and what you must avoid. Beyond this, the game is a casual experience.
  • Target Audience: This game is targeted for young audiences.
  • Play Time: Takes about two minutes to play.
